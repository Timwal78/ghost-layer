The Problem

You gave your AI agent AWS keys. Now you're watching CloudTrail at 3am.

Today's autonomous AI systems are duct-taped directly to production APIs with static credentials. Agents are ephemeral bursts of intent, yet they execute with persistent human-level access. The infrastructure hasn't caught up.

Why this matters:

  • Static credentials make long-term audit trails difficult
  • Scope creep: agents inherit permissions they don't need
  • Credential rotation is manual and error-prone
  • No cryptographic proof of intent-to-execution linkage
  • A single leaked key compromises the entire system

The Solution

GHOST inverts the model:

Human model: Log in → do stuff → log out (session persists)

Agent model: Declare intent → spawn ghost → execute → evaporate → leave residue

GHOST is infrastructure built on the premise that agents are fundamentally different from humans. Agents need temporary, scoped, rotated credentials with full cryptographic proof of execution—not persistent access keys.

Core Architecture

Five core commands orchestrate the entire execution lifecycle:

ghost spawn

Create ephemeral session with scoped, rotating credentials and ed25519 keypair.

--intent, --ttl, --scope

ghost possess

Install local HTTP proxy interceptor (localhost:9999) for agent API routing.

--agent, --session-id

ghost act

Execute pre-validated API call through proxy with residue logging.

--tool, --action, --params

ghost evaporate

Destroy session, rotate credentials, finalize cryptographic proof.

--session-id

ghost replay

Retrieve immutable, cryptographically-signed execution history.

--session-id, --format

Quickstart

pip install ghost-layer

# Spawn an ephemeral session for AI deployment
ghost spawn --intent "deploy_staging" --ttl 300 --scope aws_ec2

# Possess the agent with proxy intercept
ghost possess --agent openai://gpt-4 --session-id gh_8f2a7c3e...

# Execute a scoped action
ghost act --tool aws_ec2 --action RunInstances --session-id gh_8f2a7c3e...

# Destroy session + finalize residue
ghost evaporate --session-id gh_8f2a7c3e...

# Audit trail (cryptographically signed)
ghost replay --session-id gh_8f2a7c3e...

Data Storage & Security

Residue Store

All execution events are logged to an immutable SQLite database at ~/.ghost/residue.db:

  • Sessions table: intent, spawned_at, ttl_seconds, root_signature
  • Actions table: tool, action, params_hash, response_hash, timestamp, signature
  • Credentials log: rotation history per session

Cryptographic Proof

Every action is signed with ed25519. The root signature covers the entire session tree, making it impossible to forge or modify without detection.

Credential Lifecycle

Stage Action
Spawn Generate ed25519 keypair + HMAC token (fresh each time)
Possess Token embedded in proxy (memory-only, never persisted)
Act Token used in X-Ghost-Token header (never in URL/body)
Evaporate Key + token destroyed; session finalized with root signature
Replay Signature verified without accessing ephemeral key (public key only)

Why GHOST Matters in the x402 Ecosystem

GHOST is the infrastructure layer between autonomous AI agents and real-world APIs. It complements the x402 payment rails (NEXUS-402, 402Proof, XAH Portal) by providing the execution safety guarantees those systems require.

  • Agent Authorization: AI agents can request XRPL/Xahau payments without human key exposure
  • Audit Trail: Every autonomous action is cryptographically logged
  • Scope Control: Agents are confined to declared intents (no privilege creep)
  • Dark Pool Compatibility: SqueezeOS signals can trigger GHOST-managed autonomous trading with full residue proof
